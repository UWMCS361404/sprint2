Functions (util.py)

Imports:

  1. User

  1. parseTxt
  1. Inputs: name:string
  2. Outputs: result:list
  3. Functionality: This function takes in the name of a CSV file and then parses the file for user data and creates instances of User, which is how we create accounts, based on the information found in the file.  The function will read each line in the file from 0 to n - 1 inclusively and will append a list of accounts which is returned to the caller. This function expects that the CSV file is formatted in the following way: each entry in the file must have the username, followed that user’s password, and finally the account type, student which is recorded as a “s” or an instructor which is recorded as an “i”.  Each one of these values has a comma between itself and the next one.  This means that the number of commas on each line is n - 1 where n is equal to the number of fields in the database.  This function does not expect, and cannot compensate for a CSV file which is formatted with a line at the start of the file to name columns.  Commas cannot be included in any of the fields; ie: a password with a comma in it is not acceptable.
